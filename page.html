Three agents

Each one plays its role.

The router picks the cheapest agent that can still produce a believable response. Cowrie handles the boring stuff. The LLMs take over when context matters.

Fidelity Impact

Not all commands are dangerous.

Every command earns an FI score from 0 to 4. The score decides what gets logged, what stays in the LLM's context window, and what triggers the cloud fallback.

  • 0

    Read / Display

    ls, whoami, cat, pwd

  • 1

    Create / Install

    touch, mkdir, wget

  • 2

    Modify / Navigate

    chmod, rm, cd, echo > file

  • 3

    Service / Elevate

    sudo, systemctl, curl | bash, nmap

  • 4

    Impact / Delete

    passwd, rm -rf /, useradd, chmod 777

Memory retention formula

Retention = (0.7 × FI_weight)
          + (0.3 × Recency_weight)

FI_weight      = fi / 4
Recency_weight = 1 / (1 + age_min)

When the impactful buffer hits 11 events, the row with the lowest retention is dropped. High-FI commands survive longer — recent recon falls off first, dangerous calls stick around as context.

Rule Engine

Your rules. Your trap.

Drop a YAML file into rules/ and override anything — responses, FI weights, routing, alert thresholds. No code changes, no redeploy.

rules/crypto_miner_trap.yaml
# match wget + .sh = fake a convincing download
- match:
    command: "wget *"
    pattern: "*.sh"
    fi_min: 1
  respond:
    template: fake_download
    exit_code: 0
    delay_ms: 420
    inject: "Saving to: '{{filename}}' [12.4K]\n██ 100%"

# high entropy pipe-to-shell = escalate + alert
- match:
    entropy_gt: 4.8
    pipe_to_shell: true
  respond:
    agent: cloud
    fi_override: 4
    alert: "siem:critical"

# anything with sudo = slow the attacker down
- match:
    command: "sudo *"
  respond:
    template: fake_sudo_prompt
    delay_ms: 1200
    loop: 3  # wrong password, 3 times
⚙
Pattern matching
Glob, regex, entropy threshold, pipe detection — match exactly what you care about.
match: command / pattern / entropy_gt
✎
Response templates
Write static strings, use {{variables}}, or point to a file. Injected verbatim into the shell session.
template: / inject:
⇄
Agent override
Force a specific agent for any match — bypass the router entirely for rules you're confident about.
agent: cowrie / ondevice / cloud
◈
FI weight override
Bump or lower the impact score for a specific command pattern without touching the classifier.
fi_override: 0–4
⏱
Delay injection
Add realistic latency or loop fake prompts to keep the attacker spinning their wheels.
delay_ms: / loop:
🔔
SIEM alerts
Tag a rule with alert: and every match fires an enriched event straight to your pipeline.
alert: "siem:critical"
↻
Hot-reload
Drop a new .yaml into rules/ while the honeypot is live. It picks it up on the next command — no restart, no dropped sessions.
Integrations

Plug in your own stack.

hydraPOT ships with sensible defaults. Swap in your SIEM and your on-device LLM without touching the core.

⇌
SIEM Integration
Every command, FI score, agent decision, and raw response is emitted as a structured JSON event. Point the webhook at Splunk, Elastic, Graylog, or any custom pipeline — the format stays the same.
HYDRAPOT_SIEM_ENDPOINT=http://splunk:8088/services/collector \
python main.py

Splunk HEC · Elastic Logstash · custom webhook · stdout JSONL

◉
On-Device LLM
The ondevice_agent loader is model-agnostic. Swap Qwen for Phi-3, Mistral, Gemma, or any HuggingFace checkpoint by setting one env var — session state and the FI router wire up automatically.
ONDEVICE_MODEL=microsoft/Phi-3-mini-4k-instruct \
python main.py

Qwen2.5 · Phi-3 · Mistral · Gemma · any GGUF via llama.cpp
